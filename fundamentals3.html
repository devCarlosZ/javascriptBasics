Where do I find functions?

Functions allow you to store a pice of code that does a single task inside a defined block that you can call whenever you need using a single short commmand.

Anytime you use a Javascript structure that features a pair of -()- and you are not using a for loop, while or do...while loop, or if..else statement, you are making use of a function

Built-in browser functions

Manipulating strings with the replace() function:

          
              let myText = "I am a string";
              let newString = myText.replace('string', 'sausage');
              console.log(newString);
              // the replace() string function take a source string,
              // and a target string and replaces the soure string,
              // with the target string, and returns the newly formed string.

Manipulating an array function with the join() function: 

          
              let myArray = ['I', 'love', 'chocolate', 'frogs'];
              let madeAString = myArray.join(' ');
              console.log(madeAString);
              // the join() function takes an array, joins
              // all the array items together into a single

Use the random() function to create a random number:

      
        let myNumber = Math.random();
        // the random() function generates a random number between
        // 0 and up to but not uncluding 1, and returns that number

The Javascript language has many Built-in functions that allow you to do useful things wihtout having to write all the code. When you invoke some of these functions, you are using browser functions that could not be coded with JS. Some of those functions are built into the browser API that are sometimes written in other language (i.e. C++).

Function parameters

Some functions require parameters to be specified when you are invoking them - these are values that need to be inside the function parentheses.

Note: Parameters are sometimes called arguements, properties, or even attributes.

As an example, the browser's buitl-in Math.random() function doesn't require any parameters. When called, it always returns a random number between 0 and 1: 

      
        let myNumber1 = Math.random();

The browser's build-in string replace() function however needs two parametres - the substring to find in the main string, and the substring to replace: 

      
        let myText1 = 'I am a string';
        let newString1 = myText1.replace('string', 'sausage');

Note: When you need to specify mulitple parameters, they are sepaerated by commas.

It should also be noted that sometimes parametres are optional - you don't have to specify them. If you don't, the function, will generally adop some kind of default behavior. As an example, the array join() function's parameter's is 

    
      let myArray1 = ['I', 'love', 'chocolate', 'frogs'];
      let madeAString1 = myArray1.join(' ');
      // returns 'I love chocolate frogs'
      let madeAString1 = myArray1.join();
      // returns 'I,love,chocolate,frogs'

If no parameter is included to specify a joining/deliminating character, a comma is used by default.

Function scope and conflicts

Variables and other things defined inside functions, are inside scope of the function. Meaning, they are unreachable from code outside the functions.

The top level outside all of your functions, are called the global scoe. Values defined in the global scope are accessible from everywhere in the code.

Javascript is set up this way for security reasons. You don't want variables accessible from everywhere in the code.

For example, you have two hypothetical javascript files with the same variable name: 

        
          *script src="first.js"**/script*
          *script src="second.js"**/script*
          *script*
            greeting();
          *script*
          Asteriks were used instead of brackets for demonstrative purposes to prevent code from executing.
        
      
        
          // first.js
          var name = 'Chris';
          function greeting() {
            console.log('Hello ' + name + ': welcome to our company.');
          }
       
      
        
          // second.js
            var name = 'Zaptec';
            function greeting() {
              console.log('Our compnay is called ' + name + '.');
            }

Both functions you want to call are called greeting(), but you can only ever access the second.js file's greeting() function - it is applied to the HTML later on in the source code, so its variable and function overwrite the ones in first.js.

Keeping parts of your code locked away in functions avoids such problems, and is considered best practice.

What are return values?

Return values are just what they sound like - values returned by the function when it completes. 

      
        var myText2 = 'I am a string';
        var newString2 = myText2.replace('string', 'sausage');
        console.log(newString2);
        // the replace() string function takes a string,
        // replaces one substring with another, and returns
        // a new string with the replacement made

In the above example, the return value is being ssaved as a value of the newString2. Some functions don't return a value, such as void or undefined. Generally, the return value is used for the next stage of calculation or is the result. Whereas, the function is the intermediary step to attain the value.

Using return values in you own functions

Take into account the following random() function.

      
        function random(number) {
          return Math.floor(Math.random()*number);
        }

This could be written as follows:

      
        function random(number) {
          var result = Math.floor(Math.randrom()*number);
          return result;
        }

The first version is quicker to write and more compact. Both functions have the return stored differently; the first is stored in the return and the second in a variable named result.

Function declaration & expressions

In JavaScript, a function is not a "magical language structure", but a special kind of value.

Function declaration

      
        function sayHi() {
          alert("Hello");
        }

There is another syntax for creating a function that is call a Function Expression. 

      
        let sayHi = function() {
          alert("Hello");
        }

Here, the function is created and assigned to thevariable explicityly, like any othe rvalue. No matter how the function is defined, it's just a value store in the variable sayHi.

The eaning of these code samples is the same: "crate a function and put it into the variable sayHi".

You can even print out that value using alert.

      
        function sayHi() {
          alert("Hello");
        }
        alert(sayHi);  // shows the functoion code

Please note that the last the last line does not run the functoin, because there are no parentheses after sayHi. There are other programming languages where any mention of function causes its execution, but not in JavaScript!

In JavaScript, a function is a value, so we can deal with it as a value. The code above shows its string representation, which is the source code.

We can copy a funciton to another variable:


      function sayHi() {    // (1) create
        alert("Hello");
      }

      let func = sayHi;     // (2) copy

      func(); // Hello      // (3) run the copy (it works)!
      sayHi(); // hello     //      this still works too (why wouldn't it)

Here's what happens above in detaila;

  1. The Function Declaration (1) creates the function and puts it into the variable name sayHi.
  2. Line (2) copies it into the variable func. Please note agian: there are no parentheses after sayHi. If there were, then func = sayHi() would write the result of the call sayHi into func, not the function sayHi itself.
  3. Now the function can be called as both sayHi() and func().
Why is there a semicolon at the end?

You might wonder, why does Function Expressions have a semicolon ; at the end, but Function Declaration does not: 

            
              function sayHi() {
                // ...
              }

              let sayHi = function {
                // ..
              };

The answer is simple:

  • There's no need for ; at the endof the code blocks and syntax structures that use them like if{...}, for{...}, function f{...} etc.
  • A Function Expression is used inside the statement: let sayHi = ...;, as a value. It's not a code block, but rather an assignemnt. The semicolon ; is recommended at the end of statements, no matter what the value is. So semicolon here is not related to the Function Expression itself, it just terminates the statement.

Callback functions

Let's look at more examples of passing functions as values and using function expressions.

We'll write a function ask(question, yes, no) with three parameters

question

Text of the question

yes

Function to run if the answer is "Yes"

no

Function to run if the answer is "No"

The function should ask the question and, depending on the user's answer, call yes() or no(): 

      
        function ask(question, yes, no) {
          if(confim(question)) yes()
          else no();
        }

        function showOk() {
          alert("You agreed.");
        }

        function showCancel() {
          alert("you canceled the execution.");
        }

        //usage: functions, showOk, showCancel are passed as arguments to ask
        ask("Do you agree?", showOk, showCancel);

The arguments showOk and showCancel of ask are called callback functions or just callbacks.

The idea is that we pass a function and expect it to be "called back" if necessary. In our case, showOk becomes the callback for "yes" answer, and showCancel for "no" answer.

We can use Function Expressions to write teh same function much shorter: 

      
        function ask(question, yes, no) {
          if(confirm(question)) yes()
          else no();
        }
        ask(
          "Do you agree?",
          function() {alert("You agreed.")},
          function() {alert("You canceled the execution.");}
        );

Here, functions are declared right inside the ask(...) call. They have no name, and so are called anonymous. Such functions are not accessible outside of ask (because they are not assigned to variables), but that's just what we want here.

Such code appears in our scripts very naturally, it's in the spirit of JavaScript.

Function Expression vs Function Declaration

Let's formualte the key differences between Function Declaratoins and Expressions.

First, the syntax: how to differentiate between them in the code.

  • Function Declaration: a function, declared as a seperate statement, in the main code flow. 
            
          //Function Declaration
          function sum(a,b) {
            return a + b;
          }
  • Function Expression: a function, created inside an expression or inside another syntact construct. Here, the function is created at the right side of the "assigned expression" =: 
            
          //Function Expression
          let sum = function(a,b) {
            return a + b;
          };

The more suble difference is when a function is created by the JavaScritp engine.

A Function Expression is created when the execution reaches it and is usable only from that moment.

Once the execution flow passes to the right side of the assignemnt let sum = function... - here we go, the function is created and can be used (assigned, called, etc.) from now on.

Function Declarations are different.

A Function Declaration can be cancelled earlier than it is defined.

For Example, a global Function Declaration is visible in the whole script, no matter where it is.

That's due to internal algorithms. When JavaScript prepares to run the script, it first looks for global Function Declarations in it and it creates the functions. We can think of it as an "initialization stage."

And after all Function Declarations are processed, the code is executed. So it has to access to these functions.

For example, it works:

      
        sayHi("John");  //  Hello, John

        function sayHi3(name) {
          console.log('Hello, ${name}');
        }

The Function Declaration sahHi is created when JavaScript is preparing to start the script and is visible everywhere in it.

...If it were a Function Expression, then it wouldn't work.


      sayHi("Paul");  //  error!
      let sayHi = function(name) { //(*) no magic any more
        console.log(`Hello, ${name}`);
      };

Function Expressions are created when the execution reaches them. That would happen only in the line (*). Too late.

Another special feture of Function Declarations is their block scope.

In strict mode, when a Function Declaration is within a code block, it's visible everywhere inside that block. But not outside of it.

For instance, let's imageine that we need to declare a function welcome() depending on the age variable that we get during runtime. An then we plan to use it some time later.

If we use Function Declaration, it won't work as intended:

      
        let age = prompt("What is your age?", '');
        //conditionally declare a function
        if(age < 18) {
          function welcome() {
            alerty("Hello!");;
          }
        } else {
          function welcome(){
            alert("Greetings!");
          }
        }
        // ... use it later
        welcome(); // Error: welcome is not defined

That's becasue a Function Declaration is only visible inside a code block in which it resides.

Here' another example:

      
        let age = prompt("What is your age?", '');
        
        if age(< 18) {
          welcome()                 // \ (runs)
                                    //  |
          function welcome(){       //  |
            alert("Hello!");        //  | Function Declaration is available
          }                         //  | everywhere in the block where it's declared
                                    //  |
          welcome();                // \  (runs)

        } else {
          function welcome() {
            alert("Greetings!");
          }
        }
        // Here we're out of curly braces,
        // so we can not see Funcion Declarations made inside of them.
        
        welcome();  // Error welcome is not defined

What can we do to make welcome visible outside of if?

The correct approache would be to use a Function Expression and assign welcome to the variable that is declared outside of if and has the propert visibility.

This code works as intended:

      
        let age = prompt("What is your age?", '');
        let welcome;

        if (age < 18){

          welcome = function() {
            alert("Hello!");
          };

        } else {

          welcome = funcion() {
            alert("Greetings!");
          };
        }
        welcome();  // ok now

Or we could simplify it even further using a question mark operator ?: 

      
        let age = prompt('What is your age', '');
        
        let welcome = (age < 18) ?
          function() { alert("Hello!"); }:

        welcome(); // ok now
When to choose Function Declaration versus Function Expression?

As a rule of thumb, when we need to delcare a function, the first to consider is Fucntion Declaration syntax. It gives more freedom in how to organize our code, because we can call such functions befoer they are declared.

That's also better for readability, as it's esier to look up function f(...) {...} in the code than let f = function(...) {...}. Function Declarations are more "eye-catching".

...But if a Function Declaration does not suit us for some reason, or we need a conditional declaration (we've just seen an example), then Function Expression should be used.

Arrow Functions

There's one more very simple and concise syntax for creating functions, that's often better than Function Expressions. It's called "arrow functions", bacause it looks like this: 

        
          let func = (arg1, arg2, ... argN) => expression

...This creates a function func that has arguments arg1...argN, evaluates the expressions on the right side with their use and returns its result.

In other words, it's roughly the same as:

        
          let func = function(arg1, arg2, ...argN){
            return expression;
          };

..But much more concise.

Let's see an example:

        
          let summ = (a,b) => a + b;
          
          /* The arrow function is a shorther form of:
          
          let sum = function(a,b){
            return a + b;
          };
          */
          alert( sum(1,2)); //3

If we have only one arguement, then parentheses around parameters can be omitted, making that even shorter: 

          
            // same as
            // let double = function(n) { return n * 2 }
            let double = n => n * 2;
            alert(double(3));  // 6

If ther are no arguments, paratntheses should be empty (but they should be present): 

          
              let sayHi5 = () => alert("Hello!");
              sayHi5();

Arrow functions can be used inthe samee way as Function Expressions.

For instance, here's the rewritten example with welcome(): 

          
            let age = prompt("What is your age?", '');
            
            let welcome = (age < 18) ?
            () = alert("Hello") :
            () = alert("Greetings!");

            welcome(); // ok now

Arrow functions appear unfamiliar and not very readable at first, but htat quickly changes as the eyes get used to the structure.

They are very convenient for simple one-line actions, when we're just too lazy to write many words.

Multiline arrow functions

The examples above took arguments from the left of => and evaluated the right-side expressions with them.

Sometimes we need something a little bit more complex, like multiple expressions or statements. It is also possible, but we should enclose them in curly braces. Then use a normal return within them.

Like this:

            
              let sum = (a,b) => {
                let result = a + b;
                return result;
              };
              alert(sum(1,2));

Summary
  • Functions are values. They can be assigned, copied or declared in any place of the code.
  • If the function is declared as a aseparate statement in the main code flow, that's called a "Function Declaration".
  • If the function is created as part of an expression, it's called a "Function Expression".
  • Function Declarations are processed before the code block is executed. They are visible everywhere in the block.
  • Function Expressions are created when the execution flow reaches them.

In most cases when we need to declare a function, a Function Declaration is preferable, because it is visible prior to the declaration itself. That give us more flexibility in code organization, and is usually more readable.

So we Should use a Function Expression only when a Function Declaration is not fit for the task.

Arrow functions are handy for one-liners. They come in two flavors:

  1. Without curly braces: (...args) => expression - the right side is an expression: the function evaluates it and returns the result.
  2. With curly braces: (...args) = > { body } - brackets allow us to write mulitple statement inside the function, but we need an explicity return to return something.
